誰でも理解できる表現にする

マニュアルを書くときに大切なのは、誰が読んでも間違いなく理解できる表現を使うことです。判断に迷う言葉や、長すぎる文章は避け、読み手が理解しやすいマニュアルを作りましょう。

ここがポイント！

  1. 専門用語をわかりやすく説明する
  2. カタカナ語を日本語に置きかえる
  3. 具体的な表現を使う
  4. 一文を短くまとめる
  5. 図やイラストを活用する

専門用語をわかりやすく説明する

マニュアルには、専門用語が必要な場合があります。ただ、そのままでは読み手に伝わらないことがあるため、専門用語を使うときは、 わかりやすい言葉に言い換えたり、簡単な説明をそえたりすることが大切です。

たとえば、「インターフェース」という言葉は、ITに詳しくない人にはピンとこないかもしれません。

そんなときは、「画面のデザインや操作方法」という補足を追記すると、どんな意味なのかイメージしやすくなります。

また、専門用語をそのまま使う場合は、最初に説明を入れると理解しやすくなります。

たとえば、「API（Application Programming Interface 異なるソフトウェア同士をつなぐ仕組み）」のように、簡単な解説をつけると、読み手が迷わずに読み進めることができます。

カタカナ語を日本語に置きかえる

「エビデンス」「コンセンサス」「アサイン」など、カタカナ語（外来語）が多いと、内容が伝わりにくくなることがあります。

カタカナ語はできるだけ日本語に置きかえて、分かりやすいマニュアルにしましょう。

「エビデンス」は「証拠」「根拠」に、「コンセンサス」は「意見の一致」、「アサイン」は「わりあて」というように置き換えると理解しやすくなります。

どうしてもカタカナ語を使う必要がある場合は、かんたんな説明をそえるとよいでしょう。

たとえば、「ログイン（システムに入ること）」のように書くと、読み手にやさしい表現になります。

具体的な表現を使う

マニュアルの文章にあいまいな表現が多いと、読み手によって解釈が異なる場合があります。

「なるべく早く」「適切に対応」などの表現は、どのくらい早くするのか、どんな対応をすればよいのか読み手に判断を委ねることになり、その判断が同じになるとは限りません。

「なるべく早く報告する」ではなく、「○○日までに報告する」、「適切にチェックする」ではなく、「○○を△回確認する」と書くと、情報を正確に伝えることができます。

また、「多めに」「少し」などの感覚的な表現もできるだけ避け、「水を多めに入れる」ではなく、「水を○○ml入れる」とできるだけ数値を書くようにしましょう。

一文を短くまとめる

マニュアルでは「何をすればいいのか」を迅速に伝えることが大切です。

長い文章は、どこが大事なのか分かりにくくなり、読み手の理解を妨げます。

たとえば、「▼ボタンを押すことで、設定画面が開き、そこで必要な項目を選択して変更を加えたあと、保存ボタンを押すと設定が完了します。」という文では、一文の中に4つの動作が含まれていますね。

これを操作ごとの文章に分けると、読み手は手順をひとつずつ理解しやすくなります。

① 「▼ボタンを押すと、設定画面が開く。」

② 「必要な項目を選ぶ。」

③ 「選択後、保存ボタンを押す。」

一文は短くまとめて内容がスムーズに伝わり、分かりやすいマニュアルを作りましょう。

図やイラストを活用する

文章だけで説明しようとすると、内容が伝わりにくくなることがあります。

操作手順や仕組みの説明では、図やイラストを使うと、より直感的に理解しやすくなります。

「ボタンを押して次の画面に進む」と文章だけで書かれているより、実際の画面の画像やボタンの位置を示した図があると、伝わりやすくなります。

また、「部品の取り付け方法」を説明するときも、どの向きで取り付けるのか、どこに注意すべきかが分かるイラストがあると、ミス防止につながります。

ただし、画像が多すぎると逆に分かりにくくなることもありますので注意が必要です。

マニュアルは、読み手が迷わず理解できる表現で書くことが大切です。読み手が迷わず理解できる表現のポイントを意識して、分かりやすく、使いやすいマニュアルを作りましょう。
